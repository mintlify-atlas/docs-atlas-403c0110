This simulation models how students move between different spaces on a university campus (classroom, library, cafeteria) based on availability and comfort levels. Each student is an autonomous agent that makes decisions about where to go based on occupancy levels.

## Overview

The student movement simulation demonstrates emergent behavior in spatial distribution. Students continuously evaluate their current location and decide whether to move based on:

- Current occupancy levels (avoiding overcrowded spaces)
- Minimum stay duration (realistic behavior patterns)
- Available capacity in alternative locations
- Personal comfort thresholds

## Student Decision Logic

Each student agent follows this decision hierarchy:

```python
def decidir_siguiente_ubicacion(self, entorno):
    self.tiempo_en_ubicacion += 1
    
    # If not met minimum stay time, remain in current location
    if self.tiempo_en_ubicacion < self.tiempo_minimo_permanencia:
        return self.ubicacion_actual
    
    # Calculate current location occupancy (0 to 1)
    nivel_ocupacion_actual = entorno.obtener_nivel_ocupacion(self.ubicacion_actual)
    
    # If comfortable (< 70% full), 70% chance to stay
    if nivel_ocupacion_actual < 0.7 and random.random() < 0.7:
        return self.ubicacion_actual
    
    # Find location with lowest occupancy
    mejor_ubicacion = None
    menor_ocupacion = 1.0
    
    for ubicacion in UbicacionCampus:
        ocupacion = entorno.obtener_nivel_ocupacion(ubicacion)
        if ocupacion < menor_ocupacion:
            menor_ocupacion = ocupacion
            mejor_ubicacion = ubicacion
    
    return mejor_ubicacion
```

<Accordion title="Decision Logic Breakdown">
  **Step 1: Minimum Stay Duration**
  
  Students won't immediately leave a location. They stay for a minimum duration of 3-8 simulation steps, randomly assigned when they arrive.
  
  ```python
  self.tiempo_minimo_permanencia = random.randint(3, 8)
  ```
  
  **Step 2: Comfort Threshold**
  
  If the current location is less than 70% full, students have a 70% probability of staying, simulating contentment with uncrowded spaces.
  
  **Step 3: Optimal Location Search**
  
  When deciding to move, students evaluate all three locations and choose the one with the lowest occupancy level, creating a self-balancing distribution.
</Accordion>

## Capacity Constraints

The simulation enforces maximum capacities for each location:

```python
self.capacidades = {
    UbicacionCampus.AULA: 30,

    UbicacionCampus.CAFETERIA: 20

```

*Source: `simulacion_movimiento_estudiantes_universidad.py:168-173`*

Occupancy levels are normalized to 0-1 scale:

```python
def obtener_nivel_ocupacion(self, ubicacion):
    cantidad_actual = self.contar_estudiantes_en(ubicacion)
    capacidad_maxima = self.capacidades[ubicacion]
    return cantidad_actual / capacidad_maxima
```

*Source: `simulacion_movimiento_estudiantes_universidad.py:252-268`*

<Note>
  **Why normalize occupancy?**
  
  Normalizing to 0.0-1.0 scale allows fair comparison between locations with different capacities. A value of 0.5 means "half full" regardless of whether the space holds 20 or 30 students.
</Note>

## Simulation Update Cycle

Each simulation step follows this sequence:

```python
def actualizar_simulacion(self):
    # Phase 1: All students decide their next movement
    decisiones = []
    for estudiante in self.estudiantes:
        siguiente_ubicacion = estudiante.decidir_siguiente_ubicacion(self)
        decisiones.append((estudiante, siguiente_ubicacion))
    
    # Phase 2: Execute all movements
    for estudiante, nueva_ubicacion in decisiones:
        estudiante.moverse_a(nueva_ubicacion, self)
    
    # Phase 3: Record current state in history
    for ubicacion in UbicacionCampus:
        cantidad = self.contar_estudiantes_en(ubicacion)
        self.historial_ocupacion[ubicacion].append(cantidad)
```

*Source: `simulacion_movimiento_estudiantes_universidad.py:278-306`*

<Note>
  **Why separate decision and execution phases?**
  
  This ensures fairness - all students make decisions based on the *same* environmental state. If students moved immediately after deciding, early movers would influence later decisions unfairly.
</Note>

## Expected Behavior

When you run the simulation, you'll observe:

1. **Initial Imbalance**: Aula starts overcrowded (60% of students)
2. **Exodus Phase**: Students leave Aula seeking less crowded spaces
3. **Oscillation**: Populations fluctuate as students react to each other
4. **Equilibrium**: System eventually stabilizes near capacity-proportional distribution
5. **Continuous Adjustment**: Minor movements continue as individual stay durations expire

The final distribution typically approaches:
- **Aula**: ~40% (12/30 capacity)
- **Biblioteca**: ~33% (8/25 capacity)  
- **Cafetería**: ~30% (6/20 capacity)

This emergent equilibrium reflects the capacity ratios (30:25:20) without being explicitly programmed.
