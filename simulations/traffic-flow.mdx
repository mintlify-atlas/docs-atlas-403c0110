This simulation models vehicle behavior at a four-way urban intersection. Each vehicle is an autonomous agent that obeys traffic signals, maintains safe distances, and navigates through the intersection following realistic driving rules.

## Overview

The traffic simulation creates a dynamic intersection environment where:

- **4 traffic signals** control entry to the crossing zone
- **Vehicles** approach from all four directions
- **Collision detection** prevents vehicles from overlapping
- **Traffic light synchronization** allows only perpendicular flows simultaneously
- **Real-time metrics** track flow efficiency and congestion

### Semaphore Cycle Logic

Each traffic light cycles through three states:

```python
class EstadoSemaforo(Enum):
    VERDE    = "VERDE"     # Vehicle can proceed
    AMARILLO = "AMARILLO"  # Vehicle must stop (transition)
    ROJO     = "ROJO"      # Vehicle completely stopped

def _calcular_estado(self) -> EstadoSemaforo:
    if self.contador_ciclo < PASOS_VERDE:
        return EstadoSemaforo.VERDE
    elif self.contador_ciclo < PASOS_VERDE + PASOS_AMARILLO:
        return EstadoSemaforo.AMARILLO
    else:
        return EstadoSemaforo.ROJO
```

*Source: `simulacion_trafico_ciudad.py:84-166`*

<Accordion title="Phase Synchronization">
  **Horizontal Semaphores** (→ and ←):
  - Start at phase 0: **GREEN** initially
  - Control east-west traffic flow
  - Located at x = CENTRO_X ± ANCHO_CALLE
  
  **Vertical Semaphores** (↑ and ↓):
  - Start at phase 50: **RED** initially  
  - Control north-south traffic flow
  - Located at y = CENTRO_Y ± ANCHO_CALLE

### Vehicle Decision Hierarchy

Each frame, vehicles follow this priority sequence:

```python
def actualizar(self, lista_semaforos: list, lista_autos: list):
    # Step 1: Check for vehicle ahead
    hay_auto_adelante = self.detectar_auto_al_frente(lista_autos)
    
    # Step 2: Check traffic signal ahead
    semaforo_frontal = self.detectar_semaforo_al_frente(lista_semaforos)
    semaforo_dice_parar = False
    if semaforo_frontal is not None:
        semaforo_dice_parar = not semaforo_frontal.puede_pasar(self.sentido)
    
    # Step 3: Decide to stop or proceed
    if hay_auto_adelante or semaforo_dice_parar:
        self.esta_detenido = True
        return  # Don't move
    
    # Step 4: Move the vehicle
    self.esta_detenido = False
    delta_x, delta_y = self._calcular_delta()
    self.pos_x += delta_x
    self.pos_y += delta_y
    
    # Step 5: Check if outside grid
    if self.pos_x < -2 or self.pos_x > ANCHO_CUADRICULA + 2:
        self.fuera_de_pantalla = True
```

*Source: `simulacion_trafico_ciudad.py:389-426`*

<Accordion title="Priority Explanation">
  1. **Collision avoidance** takes highest priority - vehicles stop if another is ahead
  2. **Traffic signal obedience** comes second - stop if signal is yellow/red
  3. **Movement** only occurs if both checks pass
  4. **Cleanup** marks vehicles that exit the grid for removal
  
  This mirrors real driving: you avoid crashes first, obey signals second, then proceed.
</Accordion>

## Collision Detection

The collision detection algorithm prevents visual overlap between vehicles:

```python
def detectar_auto_al_frente(self, lista_autos: list) -> bool:
    MITAD_LARGO_AUTO = 0.5  # Half the vehicle length
    
    for otro_auto in lista_autos:
        if otro_auto.id_auto == self.id_auto:
            continue  # Don't compare with self
        
        # Calculate distance along axis of travel and lateral offset
        if self.sentido == SentidoVia.DERECHA:
            diferencia_eje = (otro_auto.pos_x - MITAD_LARGO_AUTO) - (self.pos_x + MITAD_LARGO_AUTO)
            diferencia_lateral = abs(otro_auto.pos_y - self.pos_y)
        # ... (similar logic for other directions)
        
        # Vehicle is blocking if:
        # 1. Its rear is ahead of this vehicle's front
        # 2. Gap is less than safety distance
        # 3. Both are in the same lane
        if (-0.1 < diferencia_eje < DISTANCIA_SEGURIDAD and
                diferencia_lateral < TOLERANCIA_LATERAL):
            return True  # Vehicle blocking
    
    return False  # Path clear
```

*Source: `simulacion_trafico_ciudad.py:337-387`*

<Warning>
  **Safety Distance Calculation**
  
  The algorithm compares the **rear** of the front vehicle with the **front** of the following vehicle:
  
  ```
  Front vehicle rear: otro_auto.pos_x - MITAD_LARGO_AUTO
  This vehicle front:  self.pos_x + MITAD_LARGO_AUTO
  
  Gap = Front_rear - This_front
  ```
  
  If gap < 1.8 cells, the following vehicle stops. This creates visible separation between vehicles.
</Warning>

## Semaphore Detection

Vehicles detect relevant traffic signals within a range:

```python
def detectar_semaforo_al_frente(self, lista_semaforos: list):
    semaforo_relevante = None
    distancia_minima_encontrada = float('inf')
    
    for sem in lista_semaforos:
        # Check if this semaphore controls this vehicle's direction
        if self.sentido not in sem.sentidos_controlados:
            continue
        
        # Calculate distance based on travel direction
        if self.sentido == SentidoVia.DERECHA:
            diferencia = sem.pos_x - self.pos_x
        elif self.sentido == SentidoVia.IZQUIERDA:
            diferencia = self.pos_x - sem.pos_x
        # ... (similar for vertical directions)
        
        # Only consider semaphores AHEAD and within detection range
        if 0 < diferencia < DISTANCIA_DETECCION_SEMAFORO:
            if diferencia < distancia_minima_encontrada:
                semaforo_relevante = sem
    
    return semaforo_relevante
```

*Source: `simulacion_trafico_ciudad.py:294-335`*

<Note>
  **Detection Range = 3.0 cells**
  
  Vehicles "see" traffic signals 3 cells ahead. This gives enough reaction time to stop smoothly without sudden braking at the intersection line.
</Note>

### Entry Zone Protection

Vehicles only spawn if the entry zone is clear:

```python
def _zona_entrada_libre(self, pos_x: float, pos_y: float) -> bool:
    margen_entrada = DISTANCIA_SEGURIDAD + 1.0  # Exclusion zone
    for auto_existente in self.lista_autos:
        distancia = ((auto_existente.pos_x - pos_x) ** 2 +
                     (auto_existente.pos_y - pos_y) ** 2) ** 0.5
        if distancia < margen_entrada:
            return False  # Entry blocked
    return True  # Clear to spawn
```

*Source: `simulacion_trafico_ciudad.py:532-549`*

This prevents vehicles from spawning on top of existing vehicles at the grid edges.

## Expected Behavior Patterns

When running the simulation, observe:

1. **Initial Flow**: Horizontal vehicles (blue) move freely as their signals start green
2. **Queue Formation**: Vertical vehicles (red) accumulate at red signals
3. **Signal Transition**: After 40 frames, horizontal signals turn yellow then red
4. **Flow Reversal**: Vertical signals turn green, releasing queued vehicles
5. **Cyclic Pattern**: Alternating flow between horizontal and vertical every ~50 frames
6. **Efficiency Waves**: Flow percentage oscillates with signal cycles
7. **Steady State**: After ~100 frames, system reaches equilibrium with predictable patterns

<Note>
  **Emergent Traffic Waves**
  
  You may observe "phantom traffic jams" - spontaneous slowdowns that propagate backward through queues even after signals turn green. This emergent behavior matches real traffic patterns.
</Note>
