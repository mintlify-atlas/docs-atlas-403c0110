## Student Movement Simulation

<Steps>
  <Step title="Basic Execution">
    Run the simulation with default parameters (50 students, 100 time steps):
    
    ```powershell
    python run_sim.py
    ```
    
    This launches an animated window showing:
    - **Left panel**: Spatial view of the campus with colored zones
    - **Right panel**: Real-time occupancy graph tracking student distribution
  </Step>

  <Step title="Customize Parameters">
    Adjust the number of students and simulation duration:
    
    ```powershell
    python run_sim.py --num 50 --pasos 100
    ```
    
    <ParamField path="--num" type="integer" default="50">
      Number of student agents to simulate
    </ParamField>
    
    <ParamField path="--pasos" type="integer" default="100">
      Number of time steps (animation frames) to run
    </ParamField>
  </Step>

  <Step title="Observe the Behavior">
    Watch how emergent patterns develop:
    
    <Check>Students initially cluster in the classroom (60% start there)</Check>
    <Check>Overcrowding triggers migration to less occupied spaces</Check>
    <Check>Occupancy levels gradually balance across all three locations</Check>
    <Check>Students remain in comfortable locations (< 70% capacity) with 70% probability</Check>
    
    The console displays detailed statistics:

## Traffic Intersection Simulation

<Steps>
  <Step title="Basic Execution">
    Run the traffic simulation with default grid size (20x20) and 200 time steps:
    
    ```powershell
    python run_traffic.py
    ```
    
    This opens a visualization showing:
    - **Left panel**: 4-way intersection with animated vehicles and traffic lights
    - **Top right**: Traffic flow efficiency percentage
    - **Bottom right**: Active vs. stopped vehicle counts
  </Step>

  <Step title="Customize Parameters">
    Modify the grid dimensions and simulation length:
    
    ```powershell
    python run_traffic.py --ancho 20 --alto 20 --pasos 200
    ```
    
    <ParamField path="--ancho" type="integer" default="20">
      Grid width in cells (affects visualization scale, not intersection logic)
    </ParamField>
    
    <ParamField path="--alto" type="integer" default="20">
      Grid height in cells (affects visualization scale, not intersection logic)
    </ParamField>
    
    <ParamField path="--pasos" type="integer" default="200">
      Number of simulation steps (frames) to execute
    </ParamField>
  </Step>

  <Step title="Understand Traffic Behavior">
    Observe the autonomous decision-making hierarchy:
    
    <Steps>
      <Step title="Priority 1: Collision Avoidance">
        Vehicles detect cars ahead and maintain a safety distance of 1.8 cells
        
        ```python
        hay_auto_adelante = self.detectar_auto_al_frente(lista_autos)
        ```
      </Step>
      
      <Step title="Priority 2: Traffic Light Compliance">
        Cars stop for yellow and red lights within detection range (3.0 cells)
        
        ```python
        semaforo_frontal = self.detectar_semaforo_al_frente(lista_semaforos)
        semaforo_dice_parar = not semaforo_frontal.puede_pasar(self.sentido)
        ```
      </Step>
      
      <Step title="Priority 3: Movement">
        If path is clear, vehicles advance at 0.4 cells per time step
        
        ```python
        if not (hay_auto_adelante or semaforo_dice_parar):
            delta_x, delta_y = self._calcular_delta()
            self.pos_x += delta_x
            self.pos_y += delta_y
        ```
      </Step>
    </Steps>
  </Step>

  <Step title="Monitor Traffic Light Cycles">
    Traffic lights follow synchronized phases:
    
    | Phase | Duration | Horizontal Lights | Vertical Lights |
    |-------|----------|-------------------|------------------|
    | 1     | 40 steps | 🟢 GREEN         | 🔴 RED          |
    | 2     | 10 steps | 🟡 YELLOW        | 🔴 RED          |
    | 3     | 50 steps | 🔴 RED           | 🟢 GREEN        |
    | 4     | 10 steps | 🔴 RED           | 🟡 YELLOW       |
    
    Total cycle: 100 steps (50 for each direction)
  </Step>
</Steps>

## Common Observations

<AccordionGroup>
  <Accordion title="Why do students cluster in one location?">
    If many students start in the same area and it's under 70% capacity, they'll likely stay (70% stay probability). Once capacity exceeds 70%, you'll see rapid migration to less crowded spaces.
  </Accordion>
  
  <Accordion title="Why do cars appear to 'teleport' away?">
    Vehicles that exit the grid boundaries (after crossing the intersection) are automatically removed with the `fuera_de_pantalla` flag. This prevents infinite accumulation and simulates continuous traffic flow.
  </Accordion>
  
  <Accordion title="Why don't all cars stop at red lights?">
    Only cars within **3.0 cells** of their corresponding traffic light detect it. Cars far from the intersection continue moving freely until they enter the detection zone.
  </Accordion>
  
  <Accordion title="How is collision avoidance implemented?">
    Each vehicle checks if another car's rear is within **1.8 cells** ahead in the same lane (using `TOLERANCIA_LATERAL = 0.65` to define "same lane"). If detected, the vehicle stops immediately.
  </Accordion>
</AccordionGroup>
