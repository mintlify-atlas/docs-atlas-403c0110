## Overview

The `CampusUniversitario` class represents the complete campus environment with all spaces and students. It manages capacity constraints, tracks occupancy, coordinates student movement, and provides simulation state data.

**Core Responsibilities:**
- Maintains maximum capacity for each campus space
- Counts and tracks student distribution
- Coordinates synchronized movement of all students
- Records occupancy history for analysis
- Provides statistical reports

### actualizar_simulacion

<ParamField path="actualizar_simulacion" type="method">
  Executes one time step of the simulation. This is the core simulation loop method.
  
  ```python
  def actualizar_simulacion(self)
  ```
  
  **Three-Phase Process:**
  
  **Phase 1: Decision** - All students evaluate their next move
  ```python
  decisiones = []
  for estudiante in self.estudiantes:
      siguiente_ubicacion = estudiante.decidir_siguiente_ubicacion(self)
      decisiones.append((estudiante, siguiente_ubicacion))
  ```
  
  **Phase 2: Movement** - All movements execute simultaneously
  ```python
  for estudiante, nueva_ubicacion in decisiones:
      estudiante.moverse_a(nueva_ubicacion, self)
  ```
  
  **Phase 3: Recording** - Current state saved to history
  ```python
  for ubicacion in UbicacionCampus:
      cantidad = self.contar_estudiantes_en(ubicacion)
      self.historial_ocupacion[ubicacion].append(cantidad)
  ```
  
  <ResponseField name="return" type="None">
    Method updates simulation state in-place (no return value)
  </ResponseField>
</ParamField>

**Source:** `simulacion_movimiento_estudiantes_universidad.py:278-305`

#### Why This Three-Phase Approach?

1. **Fair Decision-Making**: All students decide simultaneously based on the same state
2. **Prevents Order Bias**: First student doesn't have advantage over last
3. **Realistic Parallelism**: Simulates concurrent decision-making
4. **Clean State Recording**: History captures post-movement state

#### Example Usage

```python
# Run 10 simulation steps
for step in range(10):
    campus.actualizar_simulacion()
    stats = campus.obtener_estadisticas()
    print(f"Step {step + 1}:")
    for ubicacion, datos in stats.items():
        print(f"  {ubicacion}: {datos['cantidad']} ({datos['porcentaje']:.1f}%)")
```

### _crear_estudiantes

<ParamField path="_crear_estudiantes" type="method">
  Internal method that creates and distributes the student population.
  
  ```python
  def _crear_estudiantes(self, num_estudiantes)
  ```
  
  **Distribution Strategy:**
  - 60% in `AULA` (simulates post-class scenario)
  - 20% in `BIBLIOTECA`
  - 20% in `CAFETERIA`
  
  This creates an interesting initial state where the classroom is overcrowded, forcing dynamic redistribution.
  
  <ParamField path="num_estudiantes" type="int" required>
    Total number of students to create
  </ParamField>
</ParamField>

**Source:** `simulacion_movimiento_estudiantes_universidad.py:204-239`

#### Implementation Details

```python
# Build distribution list
ubicaciones_iniciales = (
    [UbicacionCampus.AULA] * int(num_estudiantes * 0.6) +
    [UbicacionCampus.BIBLIOTECA] * int(num_estudiantes * 0.2) +
    [UbicacionCampus.CAFETERIA] * int(num_estudiantes * 0.2)
)

# Handle rounding (ensure exact count)
while len(ubicaciones_iniciales) < num_estudiantes:
    ubicaciones_iniciales.append(random.choice(list(UbicacionCampus)))

# Shuffle to randomize creation order
random.shuffle(ubicaciones_iniciales)

# Create each student with random position in their zone
for ubicacion in ubicaciones_iniciales:
    zona = self.zonas[ubicacion]
    x = zona['x'] + random.uniform(-zona['ancho']/2, zona['ancho']/2)
    y = zona['y'] + random.uniform(-zona['alto']/2, zona['alto']/2)
    estudiante = Estudiante(ubicacion, x, y)
    self.estudiantes.append(estudiante)
```

## Complete Usage Example

```python
from enum import Enum
import random

# Create campus with 50 students
campus = CampusUniversitario(num_estudiantes=50)

# Display initial state
print("Initial Distribution:")
stats = campus.obtener_estadisticas()
for ubicacion, datos in stats.items():
    print(f"{ubicacion}: {datos['cantidad']}/{datos['capacidad']} "
          f"({datos['porcentaje']:.1f}%)")

# Run simulation for 20 steps
print("\nRunning simulation...\n")
for step in range(20):
    campus.actualizar_simulacion()
    
    # Print every 5 steps
    if (step + 1) % 5 == 0:
        print(f"Step {step + 1}:")
        stats = campus.obtener_estadisticas()
        for ubicacion, datos in stats.items():
            print(f"  {ubicacion}: {datos['cantidad']} students "
                  f"({datos['porcentaje']:.1f}%)")

# Analyze history
print("\nOccupancy History:")
for ubicacion in UbicacionCampus:
    history = campus.historial_ocupacion[ubicacion]
    avg = sum(history) / len(history)
    max_count = max(history)
    min_count = min(history)
    
    print(f"{ubicacion.value}:")
    print(f"  Average: {avg:.1f} students")
    print(f"  Range: {min_count}-{max_count} students")
```

## Design Rationale

### Why Separate Capacities Dictionary?

Allows easy configuration and experimentation:
```python
# Can be modified after creation
campus.capacidades[UbicacionCampus.AULA] = 40  # Increase classroom capacity
```

### Why Track History?

Enables:
- Time-series analysis
- Pattern recognition
- Visualization of occupancy trends
- Validation of equilibrium behavior

### Why Three-Phase Update?

Prevents race conditions and ensures fairness:
- All students see same state when deciding
- No advantage to being processed first/last
- Mimics parallel decision-making in real life
