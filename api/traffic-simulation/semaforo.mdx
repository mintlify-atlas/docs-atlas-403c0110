## Overview

The `Semaforo` class represents a traffic light that controls ONE entry lane to the intersection. Each traffic light follows an autonomous cycle: GREEN → YELLOW → RED, and can be queried by vehicles to determine whether they can proceed.

**Key responsibilities:**
- Maintains visual position for rendering
- Follows a periodic state cycle autonomously
- Provides pass/stop information to vehicles
- Synchronizes with other lights via phase offset

**Class location:** `simulacion_trafico_ciudad.py`, line 109

## Methods

### actualizar()

```python
def actualizar() -> None
```

Advances the internal counter by one step and recalculates the state. **Must be called once per animation frame.**

The modulo operator (`%`) automatically resets the counter when a complete cycle finishes, creating the perpetual GREEN → YELLOW → RED loop.

**Side effects:**
- Increments `contador_ciclo` by 1
- Wraps counter using `% duracion_ciclo`
- Updates `estado_actual` by calling `_calcular_estado()`

#### Example

```python
for semaforo in ciudad.lista_semaforos:
    semaforo.actualizar()  # Advance one step in the cycle
```

---

### puede_pasar()

```python
def puede_pasar(sentido_auto: SentidoVia) -> bool
```

Answers the question: **Can a vehicle with this direction proceed now?**

**Traffic rule:**
- Only `EstadoSemaforo.VERDE` allows passage
- `AMARILLO` and `ROJO` require stopping
- If the vehicle's direction is not in `sentidos_controlados`, the light doesn't affect it (returns `True`)

<ParamField path="sentido_auto" type="SentidoVia" required>
  The direction (`SentidoVia`) of the vehicle making the inquiry
</ParamField>

<ResponseField name="return" type="bool">
  `True` if the vehicle can proceed, `False` if it must stop
</ResponseField>

#### Example

```python
auto = Auto(5.0, 10.5, SentidoVia.DERECHA)
semaforo = ciudad.lista_semaforos[0]

if semaforo.puede_pasar(auto.sentido):
    print("Green light - proceed")
else:
    print("Red/Yellow light - stop")
    auto.esta_detenido = True
```

## Private Methods

### _calcular_estado()

```python
def _calcular_estado() -> EstadoSemaforo
```

Determines the traffic light state based on the current position in the cycle.

**Cycle division into three consecutive segments:**

```python
if contador_ciclo < PASOS_VERDE:                           # [0, 40)
    return EstadoSemaforo.VERDE
elif contador_ciclo < PASOS_VERDE + PASOS_AMARILLO:        # [40, 50)
    return EstadoSemaforo.AMARILLO
else:                                                       # [50, 100)
    return EstadoSemaforo.ROJO
```

**Returns:** The calculated `EstadoSemaforo` enum value

## EstadoSemaforo Enum

Represents the three possible states of a traffic light.

```python
class EstadoSemaforo(Enum):
    VERDE    = "VERDE"     # Vehicle can proceed freely
    AMARILLO = "AMARILLO"  # Vehicle must stop (transition state)
    ROJO     = "ROJO"      # Vehicle is fully stopped
```

**Why use Enum instead of strings or integers?**
- Prevents typos (`EstadoSemaforo.VERDE` vs error-prone `"VREDE"`)
- Makes code self-documenting
- Enables IDE autocomplete and type checking

### Usage Example

```python
if semaforo.estado_actual == EstadoSemaforo.VERDE:
    print("Proceed through intersection")
elif semaforo.estado_actual == EstadoSemaforo.AMARILLO:
    print("Prepare to stop")
else:  # ROJO
    print("Full stop required")
```

## Phase Synchronization

The `Ciudad` class creates four traffic lights with strategic phase offsets to ensure **perpendicular flows never have simultaneous green lights:**

```python
# Horizontal traffic lights (DERECHA, IZQUIERDA)
# Start at phase 0 → initially GREEN
semaforo_horizontal = Semaforo(
    pos_x=7.0, pos_y=10.5,
    sentidos_controlados=[SentidoVia.DERECHA],
    fase_inicial=0
)

# Vertical traffic lights (ARRIBA, ABAJO)
# Start at phase 50 → initially RED
fase_inicial_rojo = PASOS_VERDE + PASOS_AMARILLO  # = 50
semaforo_vertical = Semaforo(
    pos_x=9.5, pos_y=7.0,
    sentidos_controlados=[SentidoVia.ARRIBA],
    fase_inicial=fase_inicial_rojo
)
```

**Result:** When horizontal lights are green (steps 0-39), vertical lights are red (steps 50-99 of their cycle). This prevents collisions at the intersection.

## Design Pattern: Finite State Machine

The `Semaforo` class implements a **Cyclic Finite State Machine** with three states:

```
┌──────┐    40 steps     ┌──────────┐    10 steps     ┌──────┐
│ VERDE│ ─────────────> │ AMARILLO │ ─────────────> │ ROJO │
└──────┘                 └──────────┘                 └──────┘
   ▲                                                      │
   └──────────────────── 50 steps ───────────────────────┘
```

**Characteristics:**
- **Deterministic:** State transitions are predictable and time-based
- **Autonomous:** Each traffic light advances independently
- **Synchronized:** Phase offsets coordinate multiple lights without direct communication
- **Perpetual:** The cycle repeats indefinitely using modulo arithmetic
