## Overview

The `Auto` class represents an autonomous vehicle agent circulating through the intersection. Each car follows traffic rules by obeying traffic lights and avoiding collisions with other vehicles.

**Behavior characteristics:**
- Moves in a straight line according to its direction (never turns)
- Detects and obeys traffic lights ahead
- Maintains safe distance from other vehicles
- Disappears when exiting the grid boundaries

**Class location:** `simulacion_trafico_ciudad.py`, line 214

## Methods

### actualizar()

```python
def actualizar(lista_semaforos: list, lista_autos: list) -> None
```

Core decision-making and movement logic executed once per simulation step.

**Decision hierarchy (mirrors real-world driving):**
1. Check for vehicle directly ahead → Stop (highest priority)
2. Check traffic light status → Stop if YELLOW or RED
3. If unobstructed → Advance

This hierarchy ensures traffic safety in the simulation.

<ParamField path="lista_semaforos" type="list" required>
  List of all `Semaforo` objects in the city for traffic light detection
</ParamField>

<ParamField path="lista_autos" type="list" required>
  List of all `Auto` objects for collision avoidance detection
</ParamField>

**Side effects:**
- Updates `pos_x` and `pos_y` if vehicle advances
- Sets `esta_detenido` to `True` if stopped, `False` if moving
- Sets `fuera_de_pantalla` to `True` if vehicle exits boundaries

#### Example

```python
# Called by Ciudad.actualizar() for each vehicle
for auto in ciudad.lista_autos:
    auto.actualizar(ciudad.lista_semaforos, ciudad.lista_autos)
```

### detectar_semaforo_al_frente()

```python
def detectar_semaforo_al_frente(lista_semaforos: list) -> Semaforo | None
```

Searches for the relevant traffic light directly ahead within detection range.

**Detection logic:**
1. For each traffic light, verify it controls this vehicle's direction
2. Check if the light is ahead (not behind) on the correct axis
3. If within `DISTANCIA_DETECCION_SEMAFORO` (3.0 cells), it's relevant

<ParamField path="lista_semaforos" type="list" required>
  Complete list of `Semaforo` objects in the simulation
</ParamField>

<ResponseField name="return" type="Semaforo | None">
  The nearest relevant traffic light ahead, or `None` if no traffic light is within detection range
</ResponseField>

#### Example

```python
semaforo = auto.detectar_semaforo_al_frente(ciudad.lista_semaforos)
if semaforo is not None:
    puede_avanzar = semaforo.puede_pasar(auto.sentido)
    print(f"Traffic light detected: {semaforo.estado_actual.value}")
```

### detectar_auto_al_frente()

```python
def detectar_auto_al_frente(lista_autos: list) -> bool
```

Verifies if another vehicle is directly ahead within the safety distance.

**Collision avoidance is an emergent behavior:** each vehicle decides independently, but the global result is orderly traffic flow without a central controller.

**Enhanced visual separation:**
- `DISTANCIA_SEGURIDAD` set to 1.8 (> vehicle length 1.0 + margin 0.8)
- `TOLERANCIA_LATERAL` adjusted to 0.65 to avoid confusing opposite lanes
- Checks both the center and "tail" (center - half length) of other vehicles to detect overlap before it occurs visually

<ParamField path="lista_autos" type="list" required>
  Complete list of all `Auto` objects for proximity checking
</ParamField>

<ResponseField name="return" type="bool">
  `True` if a vehicle is blocking ahead, `False` if the path is clear
</ResponseField>

#### Example

```python
if auto.detectar_auto_al_frente(ciudad.lista_autos):
    print("Vehicle ahead detected - stopping")
    auto.esta_detenido = True
else:
    print("Path clear - proceeding")
```

**Returns:** `(delta_x, delta_y)` tuple:
- `DERECHA`: `(+velocidad, 0)`
- `IZQUIERDA`: `(-velocidad, 0)`
- `ARRIBA`: `(0, +velocidad)`
- `ABAJO`: `(0, -velocidad)`

## Design Pattern: Autonomous Agent

The `Auto` class exemplifies the **Agent-Based Modeling** pattern:

- **Autonomy:** Each vehicle makes its own decisions without external control
- **Local perception:** Vehicles only sense nearby traffic lights and other vehicles
- **Simple rules:** Complex traffic patterns emerge from simple individual behaviors
- **Emergent behavior:** Orderly traffic flow arises without centralized coordination

**MBA analogy:** Each vehicle is like an employee following clear policies (traffic lights) while avoiding conflicts with colleagues (collision avoidance).
